# First Driver Device Plan

We are starting with one real virtual cable path and a stable device naming contract.

## First target

- one capture endpoint: `Djehuti Router Capture`
- one monitor endpoint: `Djehuti Router Monitor`
- one simple internal route
- `48 kHz`, stereo only
- a single shared buffer between the two sides
- no effects, no mixer, no extra routing in v1

## Later expansion

- separate mic and instrument capture paths
- more named render endpoints
- better loopback and monitoring behavior
- optional virtual cable presets that match the app
